# Speedrun Android reviewer

A native Kotlin app that runs **real review sessions on the shared Anki/Speedrun
Rust engine** via JNI. It calls the same protobuf service boundary as the
desktop app (`Backend::run_service_method`), so the phone runs the exact same
core - FSRS scheduler, collection storage, and the Speedrun diagnostic engine.
No scheduling logic is reimplemented on the client.

> Status: **code-complete**. Offline review session (no sync yet - that is
> Friday). Requires the Android SDK + NDK to compile (see Prereqs).

## What it does (Wednesday MVP)

- Opens the MCAT collection at `getExternalFilesDir()/collection.anki2`.
- Pulls due cards from the shared v3 scheduler (`GetQueuedCards`), renders each
  through the engine templates (`RenderExistingCard`) in a `WebView`, and writes
  graded answers back (`AnswerCard`) so FSRS updates intervals.
- Shows the honest readiness line (`ComputeReadiness`) and follows the give-up
  rule when evidence is thin.

## Components

- `../rsandroid/` - the Rust JNI bridge (cdylib `librsandroid.so`). Depends on
  `anki` with **no TLS backend** (offline MVP; two-way sync flips on the
  `rustls` feature on Friday). `runMethod` returns a **status-tagged** byte
  array (byte 0: 1=ok, 0=error) so engine errors surface on-device.
- `app/` - the Kotlin app:
  - `NativeBackend.kt` - JNI declarations.
  - `AnkiBackend.kt` - typed protobuf wrapper + the generated (service, method)
    indices from `out/pylib/anki/_backend_generated.py`.
  - `MainActivity.kt` - the review UI (paper-like card surface + color-coded
    Again/Hard/Good/Easy).
- Protobuf: generated by Gradle (`protobuf-javalite`) straight from
  `../../proto/anki/*.proto` - there is no checked-in generated code.

## Prereqs (one-time)

- **Android SDK** (platform `android-34`, build-tools) + platform-tools (`adb`).
  Easiest via Android Studio's SDK Manager.
- **Android NDK** (r26/r27). Android Studio -> SDK Manager -> SDK Tools -> NDK
  (Side by side), or `sdkmanager "ndk;27.2.12479018"`. Then set
  `ANDROID_NDK_HOME`.
- Rust cross-compile toolchain (already set up on this machine):
  - `rustup target add --toolchain 1.92.0 aarch64-linux-android`
  - `cargo install cargo-ndk`

## Build

1. Native engine -> `jniLibs` (from `anki/rsandroid/`):

```
export ANDROID_NDK_HOME="$HOME/Library/Android/sdk/ndk/<version>"
PROTOC=$(which protoc) cargo ndk -t arm64-v8a \
  -o ../androidapp/app/src/main/jniLibs build --release
```

Produces `app/src/main/jniLibs/arm64-v8a/librsandroid.so`.

2. App: open `anki/androidapp` in Android Studio, let it sync (installs the SDK
   bits and generates the Gradle wrapper), connect the phone over USB-C (USB
   debugging enabled), and Run. Once a wrapper exists you can also use
   `./gradlew :app:installDebug`.

## Put your MCAT deck on the phone

After the first install, with the phone connected:

```
anki/tools/push_deck.sh [path/to/collection.anki2]
```

Defaults to the macOS Anki2 profile. It copies the collection into the app's
files dir (folding in any WAL), then open Speedrun to review.
